# 1. What is an Embedding Layer?

## Definition

The **Embedding Layer** is the first learnable layer of a Transformer.

Its job is to convert each Token ID into a dense numerical vector (embedding).

Think of it as a bridge between the tokenizer and the Transformer.

---

## Pipeline

```text
Raw Text

↓

Tokenizer

↓

Token IDs

↓

Embedding Layer

↓

Embedding Vectors

↓

Transformer
```

The tokenizer produces integers.

The Transformer expects vectors.

The embedding layer performs this conversion.

---

# 2. What is an Embedding Matrix?

## Definition

The embedding layer stores all embeddings in a large matrix called the **Embedding Matrix**.

Each row represents one token in the vocabulary.

Each column represents one embedding dimension.

---

## Matrix Shape

The size of an embedding matrix is:

```text
Vocabulary Size × Embedding Dimension
```

Example

Suppose

Vocabulary Size = 50,000

Embedding Dimension = 768

Then

```text
Embedding Matrix

50000 × 768
```

This means

- 50,000 rows (one for every token)
- 768 columns (features describing each token)

---

## Visual Example

```text
Token ID

↓

0  → [ ...768 values... ]

1  → [ ...768 values... ]

2  → [ ...768 values... ]

3  → [ ...768 values... ]

...

49999 → [ ...768 values... ]
```

Each row is one embedding.

---

# 3. Why is it Called a Lookup Table?

Many beginners think the model calculates embeddings every time.

It does not.

The embedding already exists inside the matrix.

The model simply retrieves it.

---

## Example

Suppose

```text
Token ID = 1254
```

The model simply reads

```text
Row 1254
```

from the embedding matrix.

Nothing is calculated.

This is why it is called a **lookup table**.

---

## Real-Time Example

Imagine a student database.

```text
Student ID

↓

1001
```

The system searches

```text
Row 1001
```

and returns the student's information.

Similarly,

```text
Token ID

↓

Row in Embedding Matrix

↓

Embedding Vector
```

---

# 4. How Does a Token ID Become an Embedding?

Suppose the tokenizer outputs

```text
"I love AI"

↓

[40, 1857, 9234]
```

The embedding layer performs

```text
40

↓

Row 40
```

```text
1857

↓

Row 1857
```

```text
9234

↓

Row 9234
```

Now the model has

```text
[
Vector 40,
Vector 1857,
Vector 9234
]
```

These vectors become the input to the Transformer.

---

# 5. Are Embeddings Calculated Every Time?

No.

This is a common misconception.

The vectors already exist inside the embedding matrix.

The only operation performed during inference is

```text
Lookup
```

Training updates these vectors,

but inference only retrieves them.

---

# 6. Why Are Embeddings Randomly Initialized?

This is one of the most important concepts.

When training starts,

the model has no knowledge of language.

Therefore,

every embedding begins with random values.

Example

```text
cat

↓

[0.12, -0.91, 0.44, ...]
```

```text
dog

↓

[-0.63, 0.08, -0.27, ...]
```

These numbers are random.

They do not contain meaning.

---

## Why Not Initialize with Zeros?

Suppose every embedding starts as

```text
[0,0,0,0]
```

Every token would look identical.

The model would receive no information to distinguish one token from another.

Random initialization breaks this symmetry,

allowing each embedding to learn independently.

---

# 7. How Do Embeddings Learn Meaning?

Initially,

all embeddings are random.

During training,

the model repeatedly predicts the next token.

After each prediction,

the model computes an error (loss).

Backpropagation calculates how each embedding contributed to that error.

The optimizer then slightly adjusts the embedding values.

This process happens billions of times.

Eventually,

similar words develop similar vectors.

---

## Example

Initially

```text
cat

↓

Random Vector
```

After training

```text
cat

↓

Meaningful Vector
```

The meaning is learned,

not manually assigned.

---

# 8. Where is the Embedding Layer Located?

The embedding layer is the very first learnable component inside the model.

Pipeline

```text
Text

↓

Tokenizer

↓

Token IDs

↓

Embedding Layer

↓

Positional Encoding

↓

Transformer Blocks

↓

Output
```

Without embeddings,

the Transformer cannot begin processing.

---

# 9. Input vs Output Embeddings

Many Transformer models have two embedding-related matrices.

---

## Input Embeddings

Used before the Transformer.

Purpose:

Convert Token IDs into vectors.

---

## Output Embeddings

Used after the Transformer.

Purpose:

Convert the Transformer's hidden representation back into probabilities over the vocabulary.

---

## Simplified Pipeline

```text
Token IDs

↓

Input Embeddings

↓

Transformer

↓

Output Embeddings

↓

Vocabulary Probabilities
```

Some models share these matrices,

while others keep them separate.

---

# 10. Why is the Embedding Layer Learnable?

Unlike the tokenizer,

the embedding layer changes during training.

Every prediction updates the embeddings.

This allows the model to gradually capture

- meaning
- similarity
- context
- relationships

between tokens.

---

# Common Misconceptions

❌ The embedding layer computes vectors from scratch every time.

✅ It performs a lookup in the embedding matrix.

---

❌ Embeddings already know language before training.

✅ They start as random values and learn through training.

---

❌ Token IDs are embeddings.

✅ Token IDs are indices used to retrieve embeddings.

---

❌ The tokenizer creates embeddings.

✅ The tokenizer creates Token IDs.

The embedding layer creates (retrieves) embedding vectors.

---

# Summary

- The embedding layer converts Token IDs into dense vectors.
- All embeddings are stored in an embedding matrix.
- The embedding matrix has the shape:
  - Vocabulary Size × Embedding Dimension
- A Token ID indexes directly into one row of the matrix.
- During inference, embeddings are retrieved through a lookup operation.
- During training, embeddings are updated using backpropagation.
- Embeddings start randomly and gradually learn semantic relationships.
- The embedding layer is the first learnable layer of a Transformer.
