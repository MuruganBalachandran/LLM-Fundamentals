# Module 1 — Tokenization Fundamentals

## Learning Goal

Understand **what tokenization is**, **why it is needed**, and **the different ways text can be split into tokens**.

---

# 1. What is Tokenization?

## Definition

**Tokenization** is the process of breaking text into smaller units called **tokens** so that an LLM can process the text.

A token can be:
- A word
- Part of a word
- A character
- A punctuation mark
- An emoji

---

## Real-Time Example

You type:

```text
Explain tokenization.
```

The LLM **does not read this sentence directly**.

Instead, it first converts it into tokens.

```text
["Explain", " token", "ization", "."]
```

These tokens are later converted into numbers (Token IDs), which the model processes.

---

# 2. Why Do LLMs Need Tokenization?

## Definition

Computers **cannot understand human language directly**.

They only process **numbers**.

Tokenization converts text into smaller pieces that can later be mapped to numbers.

---

## Real-Time Example

When you ask ChatGPT:

```text
What is Artificial Intelligence?
```

The actual process is:

```text
Your Text
      ↓
Tokenizer
      ↓
Tokens
      ↓
Token IDs
      ↓
LLM
      ↓
Response
```

The model never reads the sentence as plain English.

---

# 3. What is a Token?

## Definition

A **token** is the smallest unit of text that a tokenizer produces.

A token is **not always a complete word**.

---

## Example 1

Sentence

```text
I love AI.
```

Possible tokens

```text
["I", " love", " AI", "."]
```

---

## Example 2

Word

```text
unbelievable
```

Possible tokens

```text
["un", "believ", "able"]
```

Although it is one word, it becomes multiple tokens.

---

# 4. Types of Tokenization

There are three main approaches.

---

# A. Character-Level Tokenization

## Definition

Each character becomes one token.

---

## Example

Text

```text
ChatGPT
```

Tokens

```text
["C", "h", "a", "t", "G", "P", "T"]
```

---

## Problem

A single word becomes many tokens.

Example

```text
MachineLearning
```

becomes

```text
M a c h i n e L e a r n i n g
```

Longer sequences require more computation.

---

# B. Word-Level Tokenization

## Definition

Each word becomes one token.

---

## Example

Sentence

```text
I love machine learning
```

Tokens

```text
["I", "love", "machine", "learning"]
```

---

## Problem 1 — Huge Vocabulary

Words like

```text
play
playing
played
player
players
```

are all treated as different tokens.

The vocabulary grows very large.

---

## Problem 2 — Unknown Words (Out-of-Vocabulary)

Suppose the tokenizer has never seen

```text
HyperQuantumAI
```

Since it isn't in the vocabulary, it cannot tokenize it properly.

This is called the **Out-of-Vocabulary (OOV)** problem.

---

# C. Subword Tokenization

## Definition

Words are split into meaningful smaller pieces (subwords).

This is the approach used by modern LLMs.

---

## Example 1

```text
unbelievable
```

↓

```text
un
believ
able
```

---

## Example 2

```text
tokenization
```

↓

```text
token
ization
```

---

## Example 3

```text
playing
```

↓

```text
play
ing
```

---

## Why is this better?

A tokenizer already knows

```text
play
```

and

```text
ing
```

So even if it has never seen

```text
replaying
```

it can split it as

```text
re
play
ing
```

Instead of treating it as an unknown word.

---

# 5. Why Modern LLMs Use Subword Tokenization

Subword tokenization combines the advantages of both character-level and word-level approaches.

| Character-Level | Word-Level | Subword-Level |
|-----------------|-----------|---------------|
| Very long sequences | Huge vocabulary | Balanced vocabulary |
| Slow processing | Cannot handle unknown words | Handles unseen words efficiently |
| Harder to learn meaning | Vocabulary grows rapidly | Faster and more efficient |

---

## Real-Time Example

Suppose a new product is released.

```text
ChatGPT-10X
```

A word-level tokenizer may not recognize it.

A subword tokenizer can split it into

```text
Chat
GPT
-
10
X
```

The model can still understand it using familiar pieces.

---

# Summary

- **Tokenization** breaks text into smaller units called tokens.
- LLMs cannot process text directly; they process numbers.
- A **token** can be a word, part of a word, punctuation, or an emoji.
- **Character-level tokenization** creates long sequences.
- **Word-level tokenization** struggles with new or rare words (OOV).
- **Subword tokenization** is the standard approach because it balances vocabulary size and flexibility.
