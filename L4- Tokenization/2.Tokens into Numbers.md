# 1. What is a Tokenizer?

## Definition

A **Tokenizer** is a software component that converts human-readable text into a sequence of tokens and then maps those tokens to unique integer IDs.

It is the **first component** of every LLM pipeline.

Without a tokenizer, an LLM cannot process text.

---

## Why Do We Need a Tokenizer?

A Transformer is a neural network.

Neural networks only perform mathematical operations on numbers (vectors, matrices, tensors).

They **cannot understand characters or words directly.**

For example,

```text
Hello
```

is meaningful to humans.

To a computer, it is just five characters.

Before the Transformer can process it, it must become numbers.

That is the tokenizer's job.

---

## Real-Time Example

You type

```text
Explain Transformers.
```

The pipeline becomes

```text
User Input

↓

Tokenizer

↓

Tokens

↓

Token IDs

↓

Embedding Layer

↓

Transformer

↓

Output
```

Notice something important:

The Transformer never receives the original sentence.

It only receives numbers.

---

# 2. Is a Tokenizer an Algorithm or a Library?

This is a very common interview question.

## Answer

A tokenizer is **both**:

### A. Algorithm

An algorithm defines **how text should be split into tokens.**

Examples:

- Byte Pair Encoding (BPE)
- WordPiece
- SentencePiece

These are tokenization algorithms.

---

### B. Software

Someone has to implement those algorithms.

Libraries such as

- Hugging Face Transformers
- OpenAI's tokenizer
- SentencePiece

contain implementations of these algorithms.

Example

```python
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("gpt2")
```

Here,

AutoTokenizer is software.

Internally it executes the BPE algorithm.

---

## Interview Question

Is BPE a tokenizer?

Answer:

No.

BPE is an algorithm.

The tokenizer is software that implements BPE.

---

# 3. What is a Vocabulary?

## Definition

A vocabulary is the complete list of tokens that a tokenizer knows.

Each token appears only once.

Each token has one unique integer ID.

Think of it as a dictionary.

```text
Token

↓

Unique ID
```

---

## Example

| Token | ID |
|--------|----|
| Hello | 15496 |
| world | 995 |
| AI | 9234 |
| . | 13 |

When the tokenizer sees

```text
Hello
```

it immediately knows

```text
Hello

↓

15496
```

because that mapping already exists.

---

# 4. Who Creates the Vocabulary?

This is where many beginners get confused.

The vocabulary is **not manually written.**

It is learned automatically from a very large text corpus before the LLM is trained.

Example

Suppose we have

```text
Books

Wikipedia

News Articles

GitHub Code

Research Papers
```

The tokenizer scans billions of words.

It discovers which pieces of text occur most frequently.

Those pieces become vocabulary entries.

---

## Important

The tokenizer is trained **before** the LLM training starts.

The vocabulary is then frozen.

It usually never changes afterwards.

---

# 5. How are Token IDs Assigned?

## Definition

After the vocabulary is created,

every token receives a unique integer.

Example

```text
Hello → 15496

world → 995

AI → 9234
```

---

## Are These Numbers Meaningful?

No.

This surprises many people.

The number

```text
15496
```

does **not** mean Hello is more important than

```text
995
```

The numbers are simply unique identifiers.

Think of them like employee IDs.

Employee

```text
John

↓

ID 1203
```

Employee

```text
Alice

↓

ID 450
```

The numbers contain no meaning.

They only identify a person.

Token IDs work exactly the same way.

---

# 6. Why Can't We Use ASCII or Unicode Instead?

This is one of the most important interview questions.

Suppose we use ASCII.

Word

```text
Cat
```

ASCII

```text
C = 67

a = 97

t = 116
```

Now

```text
Dog
```

ASCII

```text
D = 68

o = 111

g = 103
```

These numbers only represent characters.

They do **not** represent language.

ASCII cannot tell us

```text
Cat

≈

Kitten
```

or

```text
Dog

≈

Puppy
```

because ASCII only stores character codes.

LLMs need token IDs because token IDs are later converted into **embeddings**, where semantic meaning is learned.

ASCII cannot do that.

---

# 7. How Does the Tokenizer Convert Text into IDs?

Suppose we input

```text
I love AI.
```

Step 1

Split into tokens.

```text
"I"

" love"

" AI"

"."
```

Step 2

Search each token in the vocabulary.

```text
I

↓

40

love

↓

1857

AI

↓

9234

.

↓

13
```

Step 3

Return

```text
[40,1857,9234,13]
```

The tokenizer's work is finished.

The Transformer now receives these numbers.

---

# 8. Does Every LLM Use the Same Tokenizer?

No.

Every model family usually has its own tokenizer.

Examples

| Model | Tokenizer |
|---------|----------------|
| GPT-2 | BPE |
| GPT-3 | Byte-level BPE |
| Llama | SentencePiece |
| BERT | WordPiece |
| T5 | SentencePiece |

That means

the same sentence

may produce different tokens

depending on the model.

---

# Common Interview Questions

### What is a tokenizer?

A software component that converts text into Token IDs using a tokenization algorithm and a vocabulary.

---

### Is a tokenizer an algorithm?

No.

Algorithms like BPE or SentencePiece define *how* tokenization works.

A tokenizer is software implementing those algorithms.

---

### Who creates the vocabulary?

The tokenizer learns it automatically from a large text corpus before LLM training.

---

### Can Token IDs change after training?

Generally no.

The vocabulary and Token IDs are fixed after tokenizer training.

---

### Why don't LLMs use ASCII?

Because ASCII only represents characters.

It does not provide a meaningful mapping for language processing or embeddings.

---

### Why are Token IDs arbitrary?

They are identifiers, not semantic values.

Meaning is learned later by the embedding layer.

---

# Common Misconceptions

❌ Token IDs contain meaning.

✅ They are only indexes.

---

❌ The tokenizer understands language.

✅ It only applies predefined rules and vocabulary lookups.

---

❌ Every LLM uses the same tokenizer.

✅ Different models use different tokenizers and vocabularies.

---

# Summary

- A tokenizer is software that converts text into Token IDs.
- It implements algorithms such as BPE or SentencePiece.
- It uses a learned vocabulary.
- Every token has a unique Token ID.
- Token IDs are arbitrary identifiers.
- Token IDs are **not** embeddings.
- The tokenizer is trained before the LLM.
- The Transformer never sees text—it only sees Token IDs.
