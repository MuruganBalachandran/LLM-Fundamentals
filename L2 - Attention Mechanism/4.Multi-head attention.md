# Recap

In the previous module, we learned about **Self-Attention**.

For every token:

* Generate Query (Q), Key (K), and Value (V)
* Compute Attention Scores
* Apply Scaling
* Apply Softmax
* Multiply Attention Weights with Values
* Produce a Context Vector

Each word finally receives a **context-aware representation**.

However, one important question remains.

> **Can a single attention mechanism capture every relationship in a sentence?**

The answer is **No**.

This is why Transformers use **Multi-Head Attention**.

---

# Why Isn't One Attention Head Enough?

Consider the sentence:

```text
The little boy kicked the football because he was excited.
```

When processing the word:

```text
he
```

The model may need to understand several relationships at the same time.

For example:

* "he" refers to **boy** (coreference)
* "kicked" is the main action (verb relationship)
* "football" is the object (semantic relationship)
* "excited" describes the subject (context relationship)

A single attention head usually focuses on **one dominant pattern**.

It cannot effectively capture every relationship simultaneously.

# Advantages of Multi-Head Attention

## 1. Learns Multiple Relationships

Different heads capture different linguistic patterns.

---

## 2. Better Context Representation

Combining multiple perspectives produces richer contextual embeddings.

---

## 3. Captures Long-Range Dependencies

One head may focus on nearby words while another focuses on distant words.

---

## 4. Improves Model Performance

Using multiple heads significantly improves translation, summarization, question answering, and text generation.

---

## 5. Enables Parallel Computation

All heads are computed simultaneously, making training efficient on GPUs.

# Multi-Head Attention vs Self-Attention

| Self-Attention                   | Multi-Head Attention                               |
| -------------------------------- | -------------------------------------------------- |
| Uses one attention mechanism     | Uses multiple attention mechanisms                 |
| Learns one dominant relationship | Learns multiple relationships simultaneously       |
| Produces one context vector      | Produces several context vectors that are combined |
| Simpler                          | More expressive and powerful                       |
| Limited representation           | Rich contextual representation                     |

# Key Takeaways

* Multi-Head Attention runs several Self-Attention mechanisms in parallel.
* Each head has its own Query, Key, and Value projection matrices.
* Different heads learn different linguistic and semantic relationships.
* Each head computes Self-Attention independently.
* Outputs from all heads are concatenated.
* A final linear projection (`WO`) combines information from all heads.
* Multi-Head Attention is one of the key innovations that makes Transformer models highly effective.
