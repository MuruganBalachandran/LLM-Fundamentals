# Recap

In the previous module, we learned:

> Attention allows a model to focus on the most relevant words instead of treating every word equally.

But one question remains:

> **How does the model know which words are important?**

The answer lies in three vectors:

* Query (Q)
* Key (K)
* Value (V)

These three vectors are the foundation of every Transformer.

---

# The Big Picture

Suppose we have the sentence:

```
The cat sat on the mat.
```

Imagine the model is currently processing the word:

```
sat
```

Its goal is to understand:

> Which other words should help me understand "sat"?

Instead of manually deciding, the Transformer computes this automatically using Query, Key, and Value.

---

# Step 1 — Start with Word Embeddings

Before Attention begins, every token is converted into an embedding vector.

Example:

```
Sentence

The
cat
sat
on
the
mat
```

↓

```
Embeddings

The → [0.42, 0.15, ...]
cat → [0.91, 0.08, ...]
sat → [0.67, 0.31, ...]
...
```

These embeddings already contain semantic meaning.

However, they are **not yet specialized for Attention**.

---

# Why Can't We Use Embeddings Directly?

Embeddings answer:

> "What does this word generally mean?"

Attention needs something different.

It needs to answer:

* Which word is asking a question?
* Which word contains useful information?
* What information should be transferred?

One vector cannot perform all three jobs effectively.

Therefore, Transformers create three new vectors from every embedding.

---

# Step 2 — Generate Query, Key, and Value

Every embedding passes through three different learned linear layers.

```
Embedding

      │
 ┌────┼────┐
 │    │    │
WQ   WK   WV
 │    │    │
 Q    K    V
```

Each matrix has its own learned parameters.

```
Query = Embedding × WQ

Key = Embedding × WK

Value = Embedding × WV
```

The weights are learned during training.

---

# Understanding Query (Q)

Think of Query as:

> **"What information am I looking for?"**

Every word creates its own Query.

Example:

Current word:

```
sat
```

Query asks:

> Which other words are useful for understanding me?

Query represents the "search request."

---

# Understanding Key (K)

Think of Key as:

> **"What information do I contain?"**

Every word generates its own Key.

The Query compares itself with every Key.

Example:

```
cat

Key:
"I represent an animal."

mat

Key:
"I represent an object."
```

---

# Understanding Value (V)

Value contains the actual information passed forward.

Think of it as:

> The knowledge stored inside each word.

Keys decide

```
Who is important?
```

Values provide

```
What information should be used?
```

---

# Human Analogy

Imagine searching for a book in a library.

Query

```
I want a book about AI.
```

Keys

```
Book 1 → Cooking

Book 2 → AI

Book 3 → History
```

Values

```
Actual contents inside each book.
```

You compare your Query with every Key.

The best matching book is selected.

Then you read its contents (Value).

This is exactly how Attention works.

---

# Step 3 — Compare Query with Every Key

Suppose we process:

```
sat
```

Its Query is compared with every Key.

```
sat(Q)

↓

The(K)

↓

cat(K)

↓

sat(K)

↓

on(K)

↓

mat(K)
```

The comparison measures similarity.

More similar

↓

Higher attention.

Less similar

↓

Lower attention.

---

# Step 4 — Compute Attention Scores

The similarity is computed using the **Dot Product**.

```
Attention Score

=

Q · K
```

A higher dot product means:

```
Higher similarity
```

Example:

| Word | Score |
| ---- | ----- |
| The  | 0.2   |
| cat  | 2.4   |
| sat  | 3.1   |
| on   | 1.2   |
| mat  | 2.1   |

These values are called **Raw Attention Scores**.

Notice that these numbers can become very large.

---

# Why Do We Scale the Scores?

When vector dimensions become large (for example 512 or 768),

the dot products also become large.

Large numbers cause Softmax to become extremely sharp.

That leads to unstable gradients during training.

To prevent this, Transformers divide the scores by:

```
√dk
```

where

```
dk = Key Dimension
```

Scaled Attention Score

```
QKᵀ
────────
 √dk
```

This makes training more stable.

---

# Step 5 — Apply Softmax

The scaled scores are still ordinary numbers.

Example

```
2.3
1.4
0.7
3.1
```

They do not represent probabilities.

Softmax converts them into probabilities.

Example

Before Softmax

```
2.3
1.4
0.7
3.1
```

↓

After Softmax

```
0.18

0.09

0.05

0.68
```

Now:

* Every value lies between 0 and 1
* All values add up to 1

These become the **Attention Weights**.

---

# Step 6 — Weight the Values

Now the model knows:

> Which words deserve more attention.

It multiplies each Value vector by its attention weight.

Example

```
cat

Weight

0.40

×

Value(cat)
```

```
mat

Weight

0.35

×

Value(mat)
```

Words with larger weights contribute more information.

Words with smaller weights contribute less.

---

# Step 7 — Sum Everything Together

Finally,

all weighted Values are added together.

```
Context Vector

=

Σ

Attention Weight

×

Value
```

This new vector becomes the representation of the current word.

Instead of only knowing

```
sat
```

the model now understands

```
sat

+

cat

+

mat

+

other relevant words
```

The representation now contains contextual meaning.

---

# Mathematical Formula

The complete Attention equation is:

```
                QKᵀ
Attention = Softmax(──────) V
               √dk
```

Where

```
Q

↓

Query Matrix
```

```
K

↓

Key Matrix
```

```

↓

Value Matrix
```

This single equation powers every Transformer model.

---

# Why This Works

Instead of remembering information sequentially,

every word asks:

> Which words are most useful for me?

The model then gathers information only from those words.

This allows Transformers to:

* Capture long-range relationships
* Understand context better
* Learn grammatical structures
* Resolve references
* Scale efficiently to very large datasets

---

# Example

Sentence

```
The animal didn't cross the road because it was tired.
```

When processing:

```
it
```

Attention gives higher weight to

```
animal
```

instead of

```
road
```

The resulting Context Vector captures the correct meaning.

---

# Key Takeaways

* Every embedding is converted into Query, Key, and Value.
* Query asks what information is needed.
* Key describes what information each word contains.
* Value stores the information that will be transferred.
* Query compares with every Key using a Dot Product.
* Scores are scaled to stabilize training.
* Softmax converts scores into probabilities.
* The probabilities weight the Value vectors.
* The weighted Values are summed into a Context Vector.
* The Context Vector becomes the new representation of the word.
