# Recap

In the previous module, we learned how the Attention mechanism works.

For every token, the Transformer creates:

* Query (Q)
* Key (K)
* Value (V)

The model then computes:

1. Attention Scores
2. Scaled Scores
3. Softmax
4. Weighted Values
5. Context Vector

One important question still remains:

> **Where do these Query, Key, and Value vectors come from?**

Do they come from another sentence?

Or from the same sentence?

The answer depends on the type of attention being used.

This brings us to **Self-Attention**.

---

# What is Self-Attention?

**Self-Attention** is a mechanism where **every token in a sequence attends to every other token in the same sequence** to understand its meaning in context.

In simple words,

Each word asks:

> **"Which other words in my own sentence should I pay attention to?"**

Unlike traditional models that process words one by one, Self-Attention allows every word to gather information from all other words simultaneously.

---

# Why is it Called "Self" Attention?

The word **Self** means the model is attending **within the same sequence**.

Example sentence:

```text
The cat sat on the mat.
```

When processing **sat**, the model can attend to:

* The
* cat
* sat
* on
* the
* mat

Notice that all these words belong to the **same sentence**.

That is why it is called **Self-Attention**.

---

# Human Analogy

Imagine you're reading the sentence:

> **The boy kicked the ball because it was heavy.**

When you read the word **"it"**, your brain automatically looks back at the previous words.

You mentally ask:

* Does "it" refer to **boy**?
* Does it refer to **ball**?

Your brain compares all the words before deciding.

Self-Attention performs this exact process mathematically.

---

# Why Do We Need Self-Attention?

Suppose we have the sentence:

```text
The animal didn't cross the road because it was tired.
```

When processing:

```text
it
```

the model must determine what **"it"** refers to.

Possible candidates:

* animal
* road

Using Self-Attention, the model assigns higher importance to:

```text
animal
```

and lower importance to:

```text
road
```

This helps the model correctly understand the sentence.

---

# Self-Attention Workflow

Consider the sentence:

```text
The cat sat on the mat.
```

Suppose we want to compute the Self-Attention output for:

```text
sat
```

The workflow is as follows.

---

# Step 1 — Convert Tokens into Embeddings

Each word becomes an embedding vector.

```text
The → Embedding

cat → Embedding

sat → Embedding

on → Embedding

the → Embedding

mat → Embedding
```

These embeddings already contain semantic information.

---

# Step 2 — Generate Query, Key, and Value

Each embedding passes through three learned linear layers.

```text
Embedding

      │
 ┌────┼────┐
 │    │    │
WQ   WK   WV
 │    │    │
 Q    K    V
```

Every token now has:

* One Query
* One Key
* One Value

---

# Step 3 — Choose the Current Token

Suppose the current token is:

```text
sat
```

Its Query vector will compare itself against the Key vectors of **every token**.

```text
sat(Q)

↓

The(K)

↓

cat(K)

↓

sat(K)

↓

on(K)

↓

the(K)

↓

mat(K)
```

---

# Step 4 — Compute Attention Scores

The model computes the similarity between:

```text
Query(sat)
```

and

```text
Key(each word)
```

using the dot product.

Example:

| Token | Score |
| ----- | ----- |
| The   | 0.4   |
| cat   | 2.8   |
| sat   | 3.1   |
| on    | 1.2   |
| the   | 0.3   |
| mat   | 2.1   |

Higher scores indicate stronger relationships.

---

# Step 5 — Scale the Scores

Large dot products can make Softmax unstable.

Therefore, every score is divided by:

```text
√dk
```

where

```text
dk
```

is the dimension of the Key vector.

This stabilizes training.

---

# Step 6 — Apply Softmax

Softmax converts raw scores into probabilities.

Example:

Before Softmax

```text
0.4
2.8
3.1
1.2
0.3
2.1
```

↓

After Softmax

| Token | Weight |
| ----- | ------ |
| The   | 0.03   |
| cat   | 0.29   |
| sat   | 0.38   |
| on    | 0.08   |
| the   | 0.02   |
| mat   | 0.20   |

These are called **Attention Weights**.

Notice:

All weights add up to **1**.

---

# Step 7 — Weight the Values

Each Value vector is multiplied by its Attention Weight.

Example:

```text
0.29 × Value(cat)

0.38 × Value(sat)

0.20 × Value(mat)
```

Words with larger weights contribute more information.

---

# Step 8 — Sum Everything

The weighted Value vectors are added together.

```text
Context Vector

=

Σ

Attention Weight × Value
```

The result becomes the new representation of:

```text
sat
```

Instead of representing only the word itself,

it now represents:

* sat
* cat
* mat
* nearby grammatical relationships
* contextual meaning

---

# Repeat for Every Token

The above process is repeated independently for **every token**.

Example:

```text
The

↓

Context Vector


cat

↓

Context Vector


sat

↓

Context Vector


on

↓

Context Vector


the

↓

Context Vector


mat

↓

Context Vector
```

At the end,

every word has a new context-aware representation.

---

Imagine drawing arrows from one word to all other words.

  ↘      ↓      ↙      ↗      ↘
      Every word attends
      to every other word

Every token can directly communicate with every other token.

---

# Why is Self-Attention Powerful?

## 1. Better Context Understanding

Each word understands its meaning using the entire sentence.

---

## 2. Long-Range Dependencies

Words that are far apart can directly interact.

Example:

```text
The book on the table near the window was expensive.
```

The word:

```text
was
```

can directly attend to:

```text
book
```

without relying on many intermediate steps.

---

## 3. Parallel Processing

Unlike RNNs,

every token computes Self-Attention simultaneously.

```text
The

↓

cat

↓

sat

↓

on

↓

mat
```

are all processed in parallel.

This is one of the biggest reasons Transformers train much faster.

---

# Why Doesn't Every Word Receive Equal Attention?

Not every word contributes equally.

Example:

```text
The cat drank the milk.
```

When processing:

```text
drank
```

the model focuses more on:

* cat
* milk

and much less on:

* the

Attention automatically learns these relationships during training.

---

# Self-Attention Formula

The complete Self-Attention equation is:

```text
                QKᵀ
Attention = Softmax(──────) V
               √dk
```

The only difference is:

**Q, K, and V all come from the same input sequence.**

This is what makes it **Self-Attention**.

---

# Attention vs Self-Attention

| Attention                                       | Self-Attention                                    |
| ----------------------------------------------- | ------------------------------------------------- |
| Query and Key may come from different sequences | Query, Key, and Value come from the same sequence |
| Used when connecting two different sequences    | Used to understand one sequence                   |
| Example: Decoder attending to Encoder           | Example: Encoder understanding a sentence         |

---

# Key Takeaways

* Self-Attention allows every token to attend to every other token in the same sequence.
* Query, Key, and Value are generated from the same input.
* Similarity is computed using the dot product.
* Softmax converts scores into attention weights.
* Weighted Value vectors create a Context Vector.
* Every token receives a new context-aware representation.
* Self-Attention enables Transformers to understand relationships between distant words while processing all tokens in parallel.
