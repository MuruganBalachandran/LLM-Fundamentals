# Where RLHF Fits in the LLM Pipeline

Modern LLMs are trained in multiple stages.

```text
Raw Text Data
      │
      ▼
Pretraining
      │
      ▼
Pretrained Model
      │
      ▼
Instruction Tuning (SFT)
      │
      ▼
Instruction-Following Model
      │
      ▼
RLHF
      │
      ▼
Aligned Assistant
      │
      ▼
Deployment
```

Think of it like learning to drive.

* **Pretraining** = Learning traffic rules.
* **Instruction Tuning** = Learning how to drive.
* **RLHF** = Becoming a safe, polite, and responsible driver through instructor feedback.

# Why RLHF Exists

Instruction Tuning teaches the model **how to answer**.

However, it does **not** teach:

* Which answer humans prefer.
* How to balance helpfulness and safety.
* How to avoid harmful responses.
* How to politely refuse unsafe requests.
* How to choose between multiple correct answers.

For many prompts, there are several acceptable responses.

Example:

User asks:

```text
Explain Machine Learning.
```

Possible responses:

Response A:

```text
Machine Learning is a branch of Artificial Intelligence that enables computers to learn patterns from data.
```

Response B:

```text
Machine Learning is when computers become smart.
```

Both are technically correct.

Humans generally prefer Response A because it is clearer and more informative.

RLHF teaches the model to prefer responses that humans consistently rate higher.

---

# Why Instruction Tuning is Not Enough

Consider this prompt:

```text
Write an email declining a job offer.
```

Possible responses:

Response A:

```text
I don't want your job.
```

Response B:

```text
Thank you very much for the opportunity. After careful consideration, I have decided to decline the offer. I truly appreciate your time and wish your team continued success.
```

Both complete the instruction.

However, humans overwhelmingly prefer Response B.

Instruction Tuning teaches **task completion**.

RLHF teaches **quality and preference**.

---

# Human Preference Data

RLHF begins by collecting human feedback.

Human evaluators compare multiple model responses to the same prompt.

Example:

Prompt:

```text
Explain gravity.
```

Model generates:

Response A

```text
Gravity is a force that attracts objects toward one another.
```

Response B

```text
Gravity is why objects fall toward Earth and why planets orbit the Sun.
```

Humans rank them:

```text
Response B ✅ Better

Response A ❌ Less Preferred
```

Notice that humans do **not** write the answer.

They simply rank which answer is better.

This ranking becomes **preference data**.

---

# Building Human Preference Data

The process looks like this:

```text
Prompt
    │
    ▼
Model Generates Multiple Answers
    │
    ▼
Human Evaluators Compare Answers
    │
    ▼
Preferred Response Selected
    │
    ▼
Preference Dataset
```

Thousands or millions of these comparisons are collected.

---

# What is a Reward Model?

A Reward Model is another neural network.

Its job is to predict:

> **"How much would humans like this response?"**

Instead of generating text,

it assigns a score.

Example:

Prompt:

```text
What is Python?
```

Response A:

```text
Python is a programming language used for web development, AI, automation, and more.
```

Reward Score:

```text
9.7
```

Response B:

```text
Python is an animal.
```

Reward Score:

```text
1.2
```

Higher score means humans are more likely to prefer that answer.

---

## How the Reward Model is Trained

Input:

```text
Prompt

Response A

Response B

Human Preference
```

Training objective:

The Reward Model learns to assign:

```text
Preferred Response → Higher Score

Less Preferred Response → Lower Score
```

Eventually, it can estimate human preferences automatically.

---

# What is the Policy Model?

The **Policy Model** is the language model that generates responses.

Initially, this is the model obtained after Instruction Tuning.

Pipeline:

```text
Pretraining
      │
      ▼
Instruction Tuning
      │
      ▼
Policy Model
```

During RLHF, this Policy Model continues learning using feedback from the Reward Model.

---

# Reinforcement Learning

Reinforcement Learning is a learning method based on **rewards**.

Instead of being told the exact correct answer,

the model receives feedback:

* Good answer → High reward
* Poor answer → Low reward

The objective becomes:

> **Generate responses that maximize the reward.**

---

## Real-World Analogy

Imagine training a dog.

Correct behavior:

🐶 Sit

Reward:

🍖 Treat

Incorrect behavior:

No reward.

Over time,

the dog learns which actions produce rewards.

RLHF works similarly.

Instead of treats,

the model receives reward scores.

---

# PPO (Proximal Policy Optimization) – High Level

Once the Reward Model can score responses,

the Policy Model is improved using **PPO**.

PPO is a Reinforcement Learning algorithm.

Its job is to update the Policy Model so it produces responses with higher reward.

---

## Simplified PPO Workflow

```text
Prompt
    │
    ▼
Policy Model Generates Response
    │
    ▼
Reward Model Scores Response
    │
    ▼
PPO Updates Policy Model
    │
    ▼
Improved Policy
```

PPO makes **small, stable updates** rather than drastic changes.

This prevents the model from forgetting what it learned during previous training stages.

---

## Why "Proximal"?

Imagine learning to ride a bicycle.

You don't completely change your riding style after every mistake.

Instead,

you make **small adjustments**.

PPO follows the same idea.

---

# Safety Alignment

Another important goal of RLHF is **alignment**.

Alignment means:

> The model's behavior matches human expectations and values.

Examples:

The model should:

* Be respectful
* Avoid dangerous advice
* Protect privacy
* Refuse malicious requests
* Give balanced information

Without alignment,

a model may generate unsafe or misleading content.

RLHF helps reduce these behaviors.

---

# Helpful, Honest, Harmless (HHH)

Many aligned AI systems aim for three important principles.

## Helpful

The model should:

* Answer questions clearly.
* Provide useful information.
* Assist users effectively.

Example:

```text
User:
Explain recursion.
```

Good response:

```text
Recursion is when a function calls itself to solve smaller versions of the same problem.
```

---

## Honest

The model should avoid pretending to know something it doesn't.

Instead of inventing information,

it should acknowledge uncertainty.

Example:

Better response:

```text
I'm not certain about that information.
```

instead of making up an answer.

---

## Harmless

The model should avoid causing harm.

Examples include:

* Refusing dangerous instructions.
* Avoiding abusive content.
* Reducing misinformation.
* Respecting privacy.

Harmless does **not** mean refusing everything.

It means responding responsibly.

# Real-World Example

Suppose the prompt is:

```text
How do I prepare for a job interview?
```

Response A:

```text
Study the company, practice common interview questions, prepare examples of your experience, and arrive on time.
```

Response B:

```text
Just be confident.
```

Humans consistently rank Response A higher.

The Reward Model learns this preference.

PPO updates the Policy Model to generate responses similar to Response A in the future.

# Limitations of RLHF

Although RLHF greatly improves LLMs, it is not perfect.

## 1. Human Preferences Can Differ

Different people may prefer different answers.

There is often no single "correct" response.

---

## 2. Expensive

Collecting human preference data requires:

* Human annotators
* Quality control
* Large-scale evaluation

This process is time-consuming and expensive.

---

## 3. Reward Hacking

Sometimes the model learns to maximize the Reward Model rather than genuinely improving.

This is called **reward hacking** or **reward exploitation**.

Researchers continuously improve training methods to reduce this problem.

---

## 4. Doesn't Eliminate Hallucinations

RLHF reduces hallucinations but cannot completely eliminate them.

The model can still generate incorrect information.

---

## 5. Cultural and Preference Bias

Human preferences vary across:

* Countries
* Languages
* Cultures
* Contexts

Designing a universally preferred assistant is challenging.

# RLHF vs Instruction Tuning

| Instruction Tuning (SFT)                 | RLHF                                               |
| ---------------------------------------- | -------------------------------------------------- |
| Learns from instruction-response pairs   | Learns from human preferences                      |
| Teaches the model to follow instructions | Teaches the model which responses humans prefer    |
| Uses labeled examples                    | Uses ranked comparisons                            |
| Focuses on task completion               | Focuses on response quality, safety, and alignment |

---

# Real-World Analogy

Imagine learning public speaking.

First,

you learn how to give a presentation.

This is **Instruction Tuning**.

Then,

your audience provides feedback:

* Speak more slowly.
* Make eye contact.
* Explain examples better.
* Be more engaging.

You improve based on that feedback.

That is **RLHF**.

# Common Misconceptions

### ❌ RLHF teaches new knowledge.

No.

Most knowledge comes from **Pretraining**.

RLHF mainly improves **behavior**, **response quality**, and **alignment**.

---

### ❌ RLHF makes the model perfect.

No.

Even after RLHF, models can still:

* Hallucinate
* Make reasoning mistakes
* Misunderstand prompts
* Reflect biases in data or feedback

---

### ❌ PPO is the only RLHF algorithm.

No.

PPO was widely used in early RLHF systems.

Today, many modern LLMs also use methods like **Direct Preference Optimization (DPO)** and other preference optimization techniques that simplify or replace parts of the traditional RLHF pipeline.

---

# Key Takeaways

* RLHF improves a model using **human preferences** rather than only labeled answers.
* Human evaluators rank model responses to create preference datasets.
* A **Reward Model** learns to predict which responses humans prefer.
* The **Policy Model** is updated to maximize reward.
* **PPO** performs stable reinforcement learning updates based on reward scores.
* RLHF aligns models to be more **Helpful, Honest, and Harmless (HHH)**.
* RLHF improves conversational quality and safety but does not eliminate hallucinations or guarantee perfect behavior.
