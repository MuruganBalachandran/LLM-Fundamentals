# 1. What is Training?

Training is the process of teaching a Large Language Model (LLM) to make better predictions by adjusting its internal parameters based on large amounts of data.

Unlike traditional software, where developers explicitly write rules, an LLM learns these rules automatically from examples.

### Definition

> **Training is the process of adjusting millions or billions of parameters so that a model becomes better at predicting the next token (word or subword).**

For example:

```text
Input:
The sky is _____

Correct Answer:
blue
```

Initially, the model might predict:

```text
pizza
```

After many updates:

```text
green
```

Eventually:

```text
blue
```

This learning process happens billions of times across enormous datasets.

### Real-World Analogy

Imagine preparing for an exam.

* Read questions.
* Attempt answers.
* Check mistakes.
* Improve.
* Repeat.

LLM training works in exactly the same way.

---

# 2. Why Can't We Hardcode Intelligence?

Traditional software works by following manually written rules.

Example:

```python
if user == "hello":
    print("Hello!")
```

This approach works for simple tasks.

However, ChatGPT can answer questions like:

* Explain quantum computing.
* Write a Python program.
* Translate Tamil to English.
* Summarize a research paper.
* Write a poem.
* Solve a math problem.

Writing rules for every possible user question is impossible.

Instead of memorizing rules, LLMs learn patterns from data.

## Hardcoded Programs vs Trained Models

| Hardcoded Program          | Trained Model                |
| -------------------------- | ---------------------------- |
| Rules written by humans    | Rules learned automatically  |
| Works only for known cases | Generalizes to unseen cases  |
| Difficult to scale         | Learns from massive datasets |
| Needs manual updates       | Learns through optimization  |

# 3. What is a Dataset?

A dataset is the collection of text that the model learns from during training.

It may contain:

* Books
* Wikipedia
* Articles
* Programming code
* Research papers
* Public websites
* Conversations
* Documentation

Example:

```text
Paris is the capital of France.

The Eiffel Tower is located in Paris.

France is in Europe.
```

The model learns relationships from these examples.

### Important

The model does **not** simply memorize the dataset.

Instead, it learns statistical patterns and relationships.

### Analogy

Dataset = Textbooks

Without textbooks, a student cannot learn.

---

# 4. What are Parameters?

Parameters are the internal numerical values inside a neural network.

They represent everything the model has learned.

People often hear:

```text
7 Billion Parameters
```

This simply means the model contains approximately seven billion adjustable numbers.

Initially:

```text
0.42
-1.18
0.007
3.29
...
```

These numbers are random.

Training continuously adjusts them until they capture useful language patterns.

### Analogy

Imagine tuning thousands of knobs on a giant sound system.

Every small adjustment improves the output.

Parameters are those knobs.

Training is the tuning process.

---

# 5. How Does the Model Learn Patterns?

The learning process is simple:

1. Read an input.
2. Predict the next token.
3. Compare prediction with the correct answer.
4. Measure the error.
5. Update parameters.
6. Repeat.

Example:

```text
Input:
I drink coffee every _____
```

Correct answer:

```text
morning
```

Prediction:

```text
elephant
```

Wrong.

Parameters are updated.

Later:

```text
night
```

Closer.

Eventually:

```text
morning
```

Correct.

The model gradually learns grammar, facts, coding patterns, reasoning patterns, and language structure.

---

# 6. What is a Loss Function?

A loss function measures how wrong the model's prediction is.

High loss = Poor prediction

Low loss = Good prediction

Example:

```text
Sentence:
The Earth revolves around the _____
```

Correct answer:

```text
Sun
```

Prediction:

```text
pizza
```

Loss is very high.

Prediction:

```text
Moon
```

Loss is lower but still incorrect.

Prediction:

```text
Sun
```

Loss is very close to zero.

### Goal

Training aims to minimize the loss over billions of examples.

# 7. How Are Parameters Updated?

Every training step follows the same cycle.

```text
Input Sentence
      │
      ▼
Predict Next Token
      │
      ▼
Calculate Loss
      │
      ▼
Find Error
      │
      ▼
Update Parameters
      │
      ▼
Repeat
```

Example:

```text
Input:
The cat sat on the _____
```

Correct answer:

```text
mat
```

Prediction:

```text
roof
```

Loss is calculated.

Parameters are updated.

Eventually:

```text
mat
```

# 8. Training vs Inference

Training and inference are completely different phases.

| Training                              | Inference                                          |
| ------------------------------------- | -------------------------------------------------- |
| Learns from data                      | Uses learned knowledge                             |
| Parameters change                     | Parameters remain fixed                            |
| Requires datasets                     | Requires user prompts                              |
| Calculates loss                       | No loss calculation                                |
| Computationally expensive             | Much faster                                        |
| Happens once during model development | Happens every time a user interacts with the model |

### Training

```text
Dataset
   ↓
Predict
   ↓
Calculate Loss
   ↓
Update Parameters
```

### Inference

```text
User Prompt
     ↓
Model
     ↓
Generate Response
```

No learning occurs during inference.

---

# 9. Complete Training Pipeline

```text
Dataset
   │
   ▼
Read Text
   │
   ▼
Predict Next Token
   │
   ▼
Compare with Correct Token
   │
   ▼
Calculate Loss
   │
   ▼
Update Parameters
   │
   ▼
Repeat Billions of Times
   │
   ▼
Trained Model
```

After training:

```text
User Prompt
      │
      ▼
Trained Model
      │
      ▼
Predict Next Token
      │
      ▼
Generate Response
```

# 10. Cricket Analogy

Imagine learning cricket.

* **Dataset:** Thousands of balls bowled during practice.
* **Parameters:** Your stance, timing, footwork, and shot selection.
* **Prediction:** Choosing how to play the next ball.
* **Loss:** Getting bowled, edging the ball, or mistiming the shot.
* **Training:** Practicing repeatedly and improving with coaching.
* **Inference:** Playing an actual match using the skills you've learned.

This is exactly how an LLM learns and later generates responses.

# Common Misconceptions

### ❌ "The model memorizes everything."

No.

It primarily learns statistical patterns and relationships.

---

### ❌ "More parameters always mean a better model."

Not necessarily.

Performance also depends on:

* Model architecture
* Dataset quality
* Training duration
* Optimization methods
* Fine-tuning

---

### ❌ "The model learns during conversations."

Generally, no.

During inference, parameters remain unchanged.

The model only uses the current conversation as context.

---

# Key Takeaways

* Training teaches an LLM by adjusting its parameters.
* Hardcoding intelligence does not scale to natural language.
* A dataset provides the examples from which the model learns.
* Parameters store learned patterns rather than explicit facts.
* The model improves by minimizing a loss function.
* Training updates parameters; inference simply uses them.
* The entire learning process is repeated billions of times to create a capable language model.
