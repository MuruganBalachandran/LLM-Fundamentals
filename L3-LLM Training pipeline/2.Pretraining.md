# What is Pretraining?

Pretraining is the **first and largest training phase** of an LLM.

During this phase, the model learns language by reading enormous amounts of text without any human-written labels.

The objective is very simple:

> **Predict the next token repeatedly until the model learns language patterns.**

Unlike supervised learning, nobody tells the model:

* This is grammar.
* This is coding.
* This is mathematics.

Instead, the model discovers these patterns automatically.

---

## Definition

> **Pretraining is the process of training a neural network on massive unlabeled text using self-supervised learning so that it becomes good at predicting the next token.**

Think of pretraining as teaching someone to read every book in a giant library.

---

# Why is Pretraining Needed?

Imagine asking a newborn baby:

> Explain quantum computing.

The baby cannot answer because it has never learned language.

Similarly, a neural network starts with **random parameters**.

Example:

```text
Parameter 1 = 0.14
Parameter 2 = -0.72
Parameter 3 = 1.83
...
```

These numbers contain **no knowledge**.

Without pretraining:

* The model cannot understand English.
* It cannot write code.
* It cannot answer questions.
* It cannot summarize text.
* It cannot translate languages.

Pretraining teaches all of these basic capabilities.

---

## Analogy

Imagine learning a new language.

Before writing essays, you first:

* Learn vocabulary
* Learn grammar
* Read books
* Understand sentence structure

That is pretraining.

Later, specialized practice (fine-tuning) improves specific skills.

# Massive Datasets

Pretraining requires enormous amounts of text.

Modern LLMs are trained on **trillions of tokens** collected from publicly available sources.

Examples include:

* Books
* Wikipedia
* Research papers
* Programming code
* News articles
* Documentation
* Educational websites
* Public discussions

The goal is diversity.

The more varied the data, the more patterns the model can learn.

---

## Example Dataset

```text
The Earth revolves around the Sun.

Python is a programming language.

Water boils at 100°C.

The capital of France is Paris.

Cats are mammals.
```

After seeing billions of similar examples, the model begins to understand relationships between words and concepts.

# What is Self-Supervised Learning?

One of the biggest misconceptions is that humans manually label every training example.

They do not.

The data already contains its own labels.

Example sentence:

```text
The cat sat on the mat.
```

Training example:

Input:

```text
The cat sat on the
```

Target:

```text
mat
```

Nobody created this label.

It comes directly from the sentence itself.

This is called **self-supervised learning**.

---

## Another Example

Sentence:

```text
I love machine learning.
```

Training pairs become:

| Input          | Target   |
| -------------- | -------- |
| I              | love     |
| I love         | machine  |
| I love machine | learning |

Every sentence automatically generates thousands of training examples.

# Next Token Prediction

This is the core objective of GPT-style models.

The model repeatedly answers one question:

> **What token comes next?**

Example:

```text
The sky is
```

Possible predictions:

| Token | Probability |
| ----- | ----------: |
| blue  |        0.93 |
| green |        0.03 |
| pizza |     0.00001 |

The correct answer is **blue**.

The closer the prediction is to the correct token, the better the model becomes.

---

## Another Example

Input:

```text
Machine learning is
```

Possible predictions:

* powerful
* useful
* fun
* important

The model learns to assign high probability to the correct continuation based on patterns in the data.

---

# Tokenization During Training

LLMs do not read raw text.

They first convert text into tokens.

Example:

```text
ChatGPT is amazing.
```

Possible tokenization:

```text
["Chat", "G", "PT", " is", " amazing", "."]
```

These tokens are then converted into IDs.

Example:

```text
[2847, 73, 291, 415, 9087, 13]
```

The neural network only sees numbers—not words.

---

## Why Tokenization?

Neural networks work with numbers, not text.

Pipeline:

```text
Sentence
      ↓
Tokenizer
      ↓
Tokens
      ↓
Token IDs
      ↓
Neural Network
```

---

# Forward Pass

The forward pass is the process of making a prediction.

Pipeline:

```text
Input Tokens
      ↓
Embedding Layer
      ↓
Transformer Layers
      ↓
Output Probabilities
```

Example:

Input:

```text
The sky is
```

Output:

```text
blue     → 0.94
green    → 0.03
red      → 0.02
pizza    → 0.00001
```

The model has not learned anything during this step.

It only predicts.

---

# Loss Calculation

After prediction, the model compares its output with the correct answer.

Example:

Input:

```text
The sky is
```

Correct token:

```text
blue
```

Prediction:

```text
pizza
```

The prediction is very wrong.

Loss is high.

If prediction is:

```text
blue
```

Loss is very low.

The loss function measures how wrong the prediction is.

The objective during training is:

> **Minimize the loss.**

---

# Backpropagation (Overview)

Once the loss is calculated, the model must determine:

> Which parameters caused this error?

Backpropagation computes how much each parameter contributed to the mistake.

Imagine a teacher grading an exam.

Instead of saying:

> Wrong.

The teacher says:

* Improve grammar.
* Improve spelling.
* Improve vocabulary.

Backpropagation provides similar feedback to every parameter.

Pipeline:

```text
Prediction
      ↓
Loss
      ↓
Backpropagation
      ↓
Parameter Gradients
```

---

# Optimizer (AdamW)

Backpropagation tells the model **what** needs to change.

The optimizer decides **how** to change it.

Modern LLMs commonly use **AdamW**.

AdamW updates billions of parameters in small, efficient steps.

Example:

Before update:

```text
Weight = 0.45
```

After one update:

```text
Weight = 0.447
```

Each update is tiny.

But after billions of updates, the model becomes highly capable.

---

## Why AdamW?

AdamW offers:

* Faster convergence
* Stable training
* Better handling of large models
* Reduced overfitting through weight decay

---

# Training Loop

Every training iteration follows the same process.

```text
Input Sentence
       ↓
Tokenization
       ↓
Forward Pass
       ↓
Prediction
       ↓
Loss Calculation
       ↓
Backpropagation
       ↓
AdamW Optimizer
       ↓
Updated Parameters
       ↓
Repeat Billions of Times
```

This loop may run for weeks or even months.

---

# Checkpoints

Training large models takes a long time.

Imagine training for two months and losing everything due to a power failure.

To avoid this, models are periodically saved.

These saved versions are called **checkpoints**.

A checkpoint contains:

* Model parameters
* Optimizer state
* Training progress
* Learning rate information

If training stops unexpectedly, it can resume from the latest checkpoint.

---

## Example

Training timeline:

```text
Day 1 → Checkpoint 1

Day 5 → Checkpoint 2

Day 10 → Checkpoint 3

Day 20 → Checkpoint 4
```

If training crashes on Day 21, it resumes from Checkpoint 4 instead of starting from scratch.

---

# Output of Pretraining

After pretraining, the model has learned:

* Grammar
* Vocabulary
* Sentence structure
* Basic reasoning patterns
* Coding syntax
* Translation patterns
* Mathematical notation
* General world knowledge present in the data

However, it is **not yet optimized to follow human instructions**.

Example:

Prompt:

```text
Tell me a joke.
```

A pretrained model might simply continue text in a generic way rather than providing a polished conversational answer.

Instruction tuning and reinforcement learning are later stages that make the model more helpful and conversational.

# Common Misconceptions

### ❌ Pretraining memorizes the internet.

No.

The model primarily learns statistical patterns, language structure, and relationships—not a perfect copy of every document.

---

### ❌ Pretraining makes ChatGPT conversational.

No.

Conversation quality mainly comes from **instruction tuning** and **RLHF**, which happen after pretraining.

---

### ❌ Pretraining happens only once.

No.

Every new model version (GPT-4, Llama 3, Gemini, etc.) undergoes its own pretraining process with updated datasets and architectures.

# Key Takeaways

* Pretraining is the foundational learning phase of every LLM.
* It uses massive unlabeled datasets and self-supervised learning.
* The model learns by repeatedly predicting the next token.
* Text is tokenized into numerical IDs before entering the neural network.
* Each forward pass produces predictions.
* The loss function measures prediction error.
* Backpropagation computes how parameters should change.
* AdamW updates the parameters efficiently.
* Checkpoints allow long training jobs to resume safely.
* The result is a pretrained model with broad language understanding, ready for fine-tuning and instruction tuning.
